# ADR-010: Role `grafana_ro` tạo bằng migration Phase2_TestQuality, mật khẩu dev mặc định

- **Bối cảnh:** Spec Phase 2 Phần 9 yêu cầu migration tạo user read-only `grafana_ro` + `GRANT SELECT` trên `TrendPoints`, `Warnings`, `MetricSets`. Nhưng Phần 2 lại quy định Phase 2 chỉ có **một migration duy nhất** tên `Phase2_TestQuality`. Việc tạo role thuộc milestone M7, trong khi migration được sinh ở M1 — nếu đợi tới M7 mới thêm thì phải sửa một migration đã apply, dễ gây lệch schema giữa các môi trường.
- **Quyết định:** (1) Đưa toàn bộ SQL tạo role + grant vào ngay migration `Phase2_TestQuality` ở M1; cả 3 bảng cần grant đều tồn tại sau migration này nên không có phụ thuộc ngược. (2) SQL viết idempotent (`IF NOT EXISTS` cho role) và dùng `format('... %I', current_database())` thay vì hardcode tên DB `eaap`, để chạy đúng cả dưới Testcontainers (tên database ngẫu nhiên). (3) Mật khẩu là giá trị dev cố định `grafana-ro-dev`, cùng tư thế với credential dev trong `docker-compose.yml` mà Phase 1 đã chấp nhận.
- **Hệ quả:** M7 chỉ còn phần docker-compose + provisioning + ghi TrendPoint + API, không đụng tới migration. Môi trường thật **bắt buộc** phải xoay vòng mật khẩu `grafana_ro` (hoặc tạo role riêng và bỏ role dev) trước khi mở Grafana ra ngoài — đây là nợ đã biết, ghi vào `docs/backlog.md`.
