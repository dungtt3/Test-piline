# ADR-012: Semgrep dùng rule vendored offline

- **Bối cảnh:** `semgrep --config auto` (và `p/...`) tải rule từ registry qua mạng. Spec Phase 3 Phần 3 nêu rõ concern này và yêu cầu chiến lược rule offline có ADR, fallback về bộ rule đóng gói sẵn khi registry không truy cập được.
- **Quyết định:** (1) `Dockerfile` cố gắng `--dump-config` các pack `p/security-audit`, `p/secrets` vào `/eaap-rules` lúc build (best-effort, `|| true`), **và** COPY một bộ rule tối thiểu tự viết trong `adapters/semgrep/rules/` (hiện có `eaap-python-sqli.yaml` bắt SQL injection nối chuỗi, kèm metadata CWE-89/OWASP-A03 để SecurityEnricher trích CWE). (2) `entrypoint.sh` mặc định `--config /eaap-rules` (offline); chỉ khi `SEMGREP_USE_REGISTRY=1` mới dùng `--config auto`. (3) Không dùng `--error` → finding không làm exit≠0; luôn đảm bảo có SARIF hợp lệ (rỗng nếu cần).
- **Hệ quả:** Job Semgrep chạy offline mặc định, deterministic theo rule đã đóng gói. Đánh đổi: coverage rule hẹp hơn `--config auto` (chỉ bộ vendored). Nâng cấp bằng cách dump full pack lúc build (có mạng) hoặc bật `SEMGREP_USE_REGISTRY=1`. E2e thật hoãn như ADR-006/011; SARIF native của Semgrep (CWE tag) đã được SecurityEnricher xử lý và test ở M1.
