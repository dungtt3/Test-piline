# ADR-007: Source của adapter .NET đặt trong `src/Eaap.Adapters.*`, Docker build context là repo root

- **Bối cảnh:** Phase 2 cần 2 adapter mode `converter` viết bằng .NET 8 (`test-report`, `coverage`). AC của spec yêu cầu **unit test** kiểm chứng logic parse trên fixture thật. Adapter Phase 1 (`megalinter`, `echo`) chỉ là shell nên nằm trọn trong `adapters/<id>/` và build với context là chính thư mục đó. Nếu nhét source .NET vào `adapters/<id>/`, test project không tham chiếu được một cách tự nhiên và solution `.slnx` sẽ có project nằm ngoài `src/`, lệch cấu trúc đã chốt.
- **Quyết định:** (1) Code adapter .NET đặt tại `src/Eaap.Adapters.<Name>/` và khai báo trong `Eaap.slnx` như mọi project khác; `tests/Eaap.UnitTests` tham chiếu trực tiếp để test logic parse. (2) `adapters/<id>/` giữ đúng 3 file contract: `Dockerfile`, `entrypoint.sh`, `manifest.yaml`. (3) Dockerfile của các adapter này build với **context là repo root** (`docker build -f adapters/<id>/Dockerfile .`) để copy được `src/Eaap.Sarif` dùng chung — tránh tự viết lại model SARIF, đúng quy tắc "SARIF qua Sarif.Sdk" trong CLAUDE.md. (4) `deploy/k3d/build-adapter.ps1|.sh` được tổng quát hoá, biết adapter nào cần context repo root.
- **Hệ quả:** Adapter contract Phần 5 không đổi (`/workspace` ro, `/results`, `/artifacts`, `/eaap-entrypoint.sh`, exit 0) — adapter Phase 1 chạy nguyên không sửa. Đổi lại, image adapter .NET phụ thuộc vào layout repo lúc build; khi tách adapter ra repo riêng sau này phải publish `Eaap.Sarif` thành NuGet nội bộ (ghi `docs/backlog.md`).
